# Configure CORS Correctly

## Goal Description
Harden Cross-Origin Resource Sharing (CORS) settings to prevent unauthorized cross-origin requests while allowing legitimate frontend interaction.

## Proposed Changes

### Backend - Main Application

#### [MODIFY] [main.ts](file:///c:/Users/m_ben/OneDrive/Escritorio/Mike/VibeIA/backend/src/main.ts)
- Update `app.enableCors` configuration.
- **Settings**:
    - `origin`: Keep `process.env.FRONTEND_URL` with fallback to localhost.
    - `methods`: Explicitly allow `GET,HEAD,PUT,PATCH,POST,DELETE,OPTIONS`.
    - `allowedHeaders`: Explicitly allow `Content-Type, Accept, Authorization`.
    - `credentials`: `true` (already present).
    - `maxAge`: Set to `3600` (1 hour) to reduce preflight requests.

## Verification Plan

### Manual Verification
1.  **Standard Request**:
    - Make a request from `http://localhost:3000` (or configured frontend).
    - Verify response headers include `Access-Control-Allow-Origin`, `Access-Control-Allow-Methods`, etc.
2.  **Unauthorized Origin**:
    - Simulate request from `http://evil.com`.
    - Verify request is blocked (CORS error in browser/Postman).
